# Portfolio Library (`portfolio-lib`)

## Project Overview

This project is a **Svelte Component Library** designed for creating beautiful, customizable, and professional portfolio websites. It converts a traditional HTML/CSS/JS portfolio layout into modern, reusable Svelte 5 components. 

The library is lightweight, framework-agnostic (within the Svelte ecosystem), responsive, and includes TypeScript support.

### Key Technologies
- **Framework:** Svelte 5 (Peer Dependency)
- **Tooling:** SvelteKit (for packaging and preview), Vite
- **Language:** JavaScript / TypeScript (declarations included)
- **Styling:** Vanilla CSS with CSS Custom Properties (Variables) for easy theming

## Architecture & Directory Structure

- `src/lib/`: The core of the library. Contains all the reusable Svelte components (`Portfolio.svelte`, `Header.svelte`, `AboutSection.svelte`, etc.).
- `src/routes/`: SvelteKit routes used as a development/preview environment to test and showcase the components.
- `static/`: Contains static assets like images and fonts used in the preview environment.
- `dist/`: The output directory where the packaged library is built (generated by `@sveltejs/package`).

### Main Components
- **`Portfolio`**: A complete, all-in-one portfolio layout that incorporates all sub-sections.
- **Individual Sections**: `Header`, `HomeSection`, `AboutSection`, `EducationSection`, `SkillsSection`, `ProjectsSection`, `BooksSection`, `AchievementsSection`, and `Footer`.

## Building and Running

The project utilizes `npm` for package management and SvelteKit scripts.

### Core Commands

- **Install Dependencies:**
  ```bash
  npm install
  ```
- **Start Development Server (Preview Site):**
  ```bash
  npm run dev
  ```
- **Build Preview Site:**
  Builds the SvelteKit application located in `src/routes` for production.
  ```bash
  npm run build
  ```
- **Package the Library:**
  Compiles the components in `src/lib` into the `dist` folder, preparing them for NPM publishing. This also generates TypeScript type definitions.
  ```bash
  npm run prepack
  ```
  *(Note: `npm run prepack` automatically runs `svelte-package` and `publint`)*

## Development Conventions

1. **Component Design:** Components should be highly customizable via props (e.g., passing custom text, URLs, or data arrays for education/skills).
2. **Styling:** Avoid hardcoding colors or styles where possible. Instead, utilize CSS custom properties defined in the `:root` pseudo-class (e.g., `--bg-color`, `--text-color`, `--sub-color`) to allow users of the library to easily override the theme.
3. **Icons & Fonts:** The library references Boxicons for icons and Google Fonts for typography. Ensure that these external dependencies are considered when rendering the components.
4. **Types:** Ensure that proper JSDoc or TypeScript definitions are maintained to provide good IDE support for library consumers.
